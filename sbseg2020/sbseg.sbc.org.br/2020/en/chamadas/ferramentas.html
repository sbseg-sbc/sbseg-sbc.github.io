IV SBSEG TOOLS SESSION


The Tools Session of the SBSeg is a meeting forum for developers and researchers interested in implementing and evaluating the performance of systems and tools that contribute to the advancement of knowledge in the area of information and computational systems security. In this tools session there will be emphasis on demonstrating new systems and tools, experiences of their development, issues of deployment and uses, as well as the perspectives created by them. Therefore, all tools accepted must be presented (demonstrated) at the SBSeg by one of the authors enrolled in the event.

  • Deadline for submission of articles:July 17, 2020 August 03, 2020

INSTRUCTIONS TO AUTHORS

The tools must be presented in a paper having the following info:

  • Description and motivation of the problem solved by the tool;
  • Architecture and description of the main features;
  • Tool URL (the source code and the license must be present, not only the binary);
  • Documentation URL, including installation instructions and system requirements;
  • Description for the planned activities for the tools session, detailing which resources and equipment are needed;
  • (optional) URL of a video detailing installation and the tool features.

The papers may be written in English or Portuguese; The formatting must comply with SBC's template for papers, available at SBC's site. Papers are limited to a maximum of 8 pages, including figures, tables, references, and appendixes. Paper submission will be only accepted in PDF format, through SBC’s JEMS System.

The evaluation process will take into account the quality, relevance and innovation of the practical results, the documentation, the submitted paper and the demonstration proposed. In this edition, the tools session will consist only of tools demonstrations.

ATTENTION: This year the ‘Tool Session’ has two types of submissions: normal and simplified.

  • Simplified submission: It should be used if the tool/dataset results from any project that have an accepted paper in a conference (or in a journal) that is Qualis scored (see here for a list of conferences and journals allowed). The author must also submit a prove the acceptance.

  • Normal submission: It should be used for any other submission that does not fit in the simplified one.

A committee will select the best tools of all the tools accepted and presented in the tools session.
